## 🪜 步骤详情

### S1 · 配置 LLM API Key ✅

**目标**：填入你的 LLM 接口信息，让应用能调用大模型

**操作**：编辑 `conf/app_config.yaml` 中的 `llm` 部分：
```yaml
llm:
  model_name: 你的模型名（如 deepseek-chat）
  api_key: 你的API Key
  base_url: 你的接口地址（如 https://api.deepseek.com/v1）
```

**验证**：填写完成后告诉我，无需运行命令。

**状态**：✅ 完成

### S2 · 启动基础设施 ✅

**目标**：用 Docker 启动 MySQL、Qdrant、Elasticsearch 三个服务

**命令**：

| `no configuration file provided` | 不在项目目录 | `cd /Users/ys/code/git/nl2sql-agent` |

**状态**：待开始

### S5 · 启动 Embedding 服务 ✅

**目标**：启动本地 Embedding 模型服务（BAAI/bge-large-zh-v1.5），供向量化使用

> ⚠️ 已完成 OPT-1 优化，使用原生 Python 服务替代 Docker 容器（Apple Silicon 适配）

**命令**：

**预期输出**：

**常见报错**：

| 报错关键词 | 原因 | 修复命令 |

| `ModuleNotFoundError: sentence_transformers` | 依赖未装 | `uv add sentence-transformers torch` |

**注意**：此服务需保持运行，**新开一个终端**继续执行后续步骤。

## 🚀 日常启动 / 停止

**一键启动**（自动检查端口，弹出新终端窗口）：
```bash
cd /Users/ys/code/git/nl2sql-agent && ./start.sh
```

**停止脚本**：

| 脚本 | 作用 |
|---|---|
| `./stop.sh` | 停止全部（前端/后端/Embedding + Docker） |
| `./stop_services.sh` | 只停前端/后端/Embedding，保留 Docker |
| `./stop_infra.sh` | 只停 Docker 基础设施 |

访问地址：`http://localhost:5173`

---

**手动启动（分步）**：
```bash
# 终端1：基础设施
cd /Users/ys/code/git/nl2sql-agent && docker compose up -d

# 终端2：Embedding 服务
cd /Users/ys/code/git/nl2sql-agent && uv run uvicorn docker.embedding.server:app --port 8081

# 终端3：后端
cd /Users/ys/code/git/nl2sql-agent && uv run uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 终端4：前端
cd /Users/ys/code/git/nl2sql-agent/data-agent-fronted/data-agent-fronted && npm run dev
```
