## S5 · 启动 Embedding 服务

**状态**：✅ 完成（OPT-1 已优化，2026-05-24）

### 问题
当前用 Docker 容器 `ghcr.io/huggingface/text-embeddings-inference:cpu-1.8` 运行，但该镜像是 amd64，在 Apple Silicon（arm64）Mac 上需要 Rosetta 模拟，启动慢、性能差。

### 候选方案
**方案A（当前）**：继续用 Docker 模拟运行，不改代码，但性能差。

**方案B（推荐）**：用原生 Python 起一个 HTTP 服务监听 8081，直接加载本地模型文件 `/Users/ys/e/models/bge-large-zh-v1.5`，绕过 Docker emulation，arm64 原生运行速度快。需要写一个小的 FastAPI server，替换 Docker 容器。

### 可行性结论
- 应用通过 `HuggingFaceEndpointEmbeddings` 调用 `POST /embed`，与 TEI 格式一致
- PyTorch 有 Apple Silicon 原生包，sentence-transformers 支持 arm64
- 方案B完全可行，实现：一个 FastAPI server 加载本地模型实现 `POST /embed`，用 `uv run` 直接跑

## S6 · 构建元数据知识库

**状态**：✅ 完成（2026-05-24）

### 执行过程

**第一次尝试**：未传配置文件参数
```bash
uv run python -m app.scripts.build_meta_knowledge
```
报错：`TypeError: argument should be a str ... not 'NoneType'`
修复：需要加 `-c conf/meta_config.yaml` 参数，且应传 `meta_config.yaml` 而非 `app_config.yaml`

**第二次尝试**：缺少 greenlet 模块
报错：`No module named 'greenlet'`
修复：`uv add greenlet`

**第三次尝试**：atguigu 用户无 dw 数据库权限
报错：`Access denied for user 'atguigu'@'%' to database 'dw'`
修复：`docker exec -it nl2sql-agent-mysql mysql -uroot -proot123 -e "GRANT ALL PRIVILEGES ON dw.* TO 'atguigu'@'%'; FLUSH PRIVILEGES;"`

**第四次尝试**：dw 数据库表不存在
报错：`Table 'dw.dim_region' doesn't exist`
原因：init SQL 只创建了 dw 数据库，未建表
修复：新建 `docker/mysql/init/02_create_dw_tables.sql`，手动执行创建 5 张 DW 表并插入示例数据

**第五次尝试**：Decimal 类型无法 JSON 序列化
报错：`TypeError: Object of type Decimal is not JSON serializable`
修复：在 `app/repositories/mysql/meta/mappers/column_info_mapper.py` 的 `to_model` 方法中将 Decimal 转为 float

**第六次尝试**：meta 数据库表名不匹配
报错：`Table 'meta.column_info' doesn't exist`
原因：init SQL 建的是 `columns/tables/metrics`，代码期望 `column_info/table_info/metric_info/column_metric`
修复：直接在 MySQL 中执行建表 SQL，创建 4 张正确命名的表

**第七次尝试**：成功
```
元数据知识库构建完成
```
