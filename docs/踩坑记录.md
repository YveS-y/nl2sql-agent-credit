# 踩坑记录

## 1. `.venv` 复制导致 uvicorn 无法启动

**现象**

```
error: Failed to spawn: `uvicorn`
  Caused by: No such file or directory (os error 2)
```

或直接调用时：

```
zsh: .venv/bin/uvicorn: bad interpreter: /Users/ys/code/git/nl2sql-agent/.venv/bin/python: no such file or directory
```

**原因**

`.venv` 是从 `nl2sql-agent` 项目复制过来的，`.venv/bin/` 下所有脚本的 shebang 行都硬编码了原项目的 Python 路径，在新项目中失效。

**解决方法**

删除损坏的 `.venv`，用 `uv sync` 按 `uv.lock` 重新创建：

```bash
rm -rf .venv && uv sync
```

**教训**

虚拟环境不能跨项目复制。新项目始终用 `uv sync` 创建环境。

## 2. `build_meta_knowledge.py` 报 `ConfigKeyError`

**现象**

```
omegaconf.errors.ConfigKeyError: Key 'layer' not in 'TableConfig'
omegaconf.errors.ConfigKeyError: Key 'examples' not in 'ColumnConfig'
omegaconf.errors.ConfigKeyError: Key 'ambiguous' not in 'MetricConfig'
```

**原因**

`conf/meta_config.yaml` 里为信贷场景新增了字段（`layer`、`examples`、`ambiguous`），但 `app/conf/meta_config.py` 里的 dataclass 没有同步更新，OmegaConf merge 时严格校验，发现多余字段就报错。

**解决方法**

在 `app/conf/meta_config.py` 对应 dataclass 里补全字段：

```python
@dataclass
class ColumnConfig:
    ...
    examples: Optional[list] = None   # 新增

@dataclass
class TableConfig:
    ...
    layer: Optional[str] = None       # 新增

@dataclass
class MetricConfig:
    ...
    ambiguous: bool = False           # 新增
```

**教训**

修改 yaml 配置新增字段后，必须同步更新对应的 Python dataclass，否则 OmegaConf 会拒绝合并。

## 3. 召回节点全部报 `Connection error.`（VPN Fake IP 拦截）

**现象**

三个召回节点（召回字段 / 召回指标 / 召回字段取值）在约 31 秒后同时失败：

```
召回指标信息失败: Connection error.
召回字段信息失败: Connection error.
召回字段取值失败: Connection error.
```

**排查过程**

1. 基础设施（MySQL / Qdrant / ES）正常 ✅
2. Embedding 服务正常，`aembed_query` 单独测试可用 ✅
3. 加完整堆栈日志后发现是 `openai.APIConnectionError` → `httpx.ConnectError`
4. `curl dashscope.aliyuncs.com` 返回 `000`（无响应），确认是网络连不通
5. DNS 解析发现 `dashscope.aliyuncs.com` → `198.18.1.141`（Clash Fake IP 段）
6. TLS 握手卡住超时——VPN 处于 TUN/Fake IP 模式，拦截了请求但未正确转发

**根本原因**

VPN 工具（ClashX 等）的 TUN 模式劫持了所有 DNS，返回假 IP。`dashscope.aliyuncs.com` 没有命中代理规则，导致连接被拦截后无法完成 TLS 握手，最终超时。

**解决方法（待完成）**

在 VPN 工具的规则配置里，给 `dashscope.aliyuncs.com` 加一条 `DIRECT` 直连规则（dashscope 是国内地址，不需要走代理）。

不同工具加规则方式：
- **ClashX**：在 `config.yaml` 的 `rules` 里加 `- DOMAIN,dashscope.aliyuncs.com,DIRECT`
- **Surge**：在 `[Rule]` 里加 `DOMAIN,dashscope.aliyuncs.com,DIRECT`

**代码层面已做的改动**

在 `app/agent/llm.py` 新增两套 LLM 初始化方案，通过 `conf/app_config.yaml` 的 `llm.bypass_proxy` 开关切换：

```yaml
llm:
  bypass_proxy: true   # true=绕过系统代理直连，false=走系统代理
```

```python
# bypass_proxy=true 时（直连，不走系统代理）
llm = ChatOpenAI(
    ...,
    http_client=httpx.Client(trust_env=False),
    http_async_client=httpx.AsyncClient(trust_env=False),
)

# bypass_proxy=false 时（走系统代理，原始写法）
llm = init_chat_model(model=..., model_provider="openai", ...)
```

**验证命令**

VPN 规则加好后，用以下命令测试 LLM 是否可用：

```bash
PYTHONPATH=. uv run python test_llm.py
```

成功输出"耗时 X 秒 + 回复内容"即为正常。
