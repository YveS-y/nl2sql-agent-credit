# 信贷数据库表结构设计

> 供 B1（建表SQL）和 B3（meta_config.yaml）参考。

## 业务背景

- 产品：消费分期12期 / 消费分期6期 / 小额消费贷3期
- 渠道：自营APP / 京东金融 / 抖音 / 线下门店-北京 / 线下门店-上海
- 数据：6个月历史，约30000笔放款记录

---

## 表清单

### DIM 层（维度表）

#### dim_product_type — 产品类型维度
| 字段 | 类型 | 说明 | 枚举值 |
|------|------|------|--------|
| product_type_id | INT PK | 产品ID | |
| product_name | VARCHAR(100) | 产品名称 | 消费分期12期/消费分期6期/小额消费贷3期 |
| term_months | INT | 还款期数 | 12/6/3 |
| annual_rate | DECIMAL(5,4) | 年化利率 | 0.1800/0.1600/0.2400 |
| max_loan_amount | DECIMAL(10,2) | 最高放款金额 | 50000/30000/10000 |

#### dim_channel — 渠道维度
| 字段 | 类型 | 说明 | 枚举值 |
|------|------|------|--------|
| channel_id | INT PK | 渠道ID | |
| channel_name | VARCHAR(100) | 渠道名称 | 自营APP/京东金融/抖音/线下门店-北京/线下门店-上海 |
| channel_type | VARCHAR(50) | 渠道类型 | 线上/线下 |

#### dim_date — 时间维度
| 字段 | 类型 | 说明 |
|------|------|------|
| date_id | INT PK | 日期ID（yyyyMMdd格式） |
| year | INT | 年份 |
| month | INT | 月份 |
| quarter | INT | 季度 |
| day | INT | 日 |

### DWD 层（明细层）

#### dwd_loan — 放款事实表（核心，30000条）
| 字段 | 类型 | 说明 | 备注 |
|------|------|------|------|
| loan_id | BIGINT PK | 放款唯一ID | |
| borrower_id | BIGINT | 借款人ID | |
| product_type_id | INT FK | 产品类型 | 关联dim_product_type |
| channel_id | INT FK | 放款渠道 | 关联dim_channel |
| loan_date_id | INT FK | 放款日期 | 关联dim_date |
| loan_amount | DECIMAL(10,2) | 放款金额 | 核心指标 |
| term_months | INT | 还款期数 | 3/6/12 |
| monthly_payment | DECIMAL(10,2) | 每月应还金额 | |
| loan_status | VARCHAR(20) | 贷款状态 | 还款中/已结清/逾期/坏账 |

#### dwd_repayment — 还款记录表
| 字段 | 类型 | 说明 |
|------|------|------|
| repayment_id | BIGINT PK | 还款记录ID |
| loan_id | BIGINT FK | 关联放款ID |
| period | INT | 第几期（1-12） |
| due_date_id | INT FK | 应还款日期 |
| due_amount | DECIMAL(10,2) | 应还金额 |
| actual_date_id | INT FK | 实际还款日期（NULL=未还） |
| actual_amount | DECIMAL(10,2) | 实际还款金额（NULL=未还） |
| status | VARCHAR(20) | 状态：已还/逾期/未到期 |

#### dwd_overdue — 逾期记录表
| 字段 | 类型 | 说明 |
|------|------|------|
| overdue_id | BIGINT PK | 逾期记录ID |
| loan_id | BIGINT FK | 关联放款ID |
| period | INT | 逾期期数 |
| overdue_days | INT | 逾期天数 |
| overdue_amount | DECIMAL(10,2) | 逾期金额 |
| overdue_date_id | INT FK | 逾期日期 |
| bucket | VARCHAR(10) | 逾期桶：M1/M2/M3+ |

## 写 meta_config.yaml 的注意事项

1. **description 要写业务含义**，不是字段名翻译：
   - ❌ `description: 渠道ID`
   - ✓ `description: 放款渠道的唯一标识，关联dim_channel维度表`

2. **alias 要尽量多**（用户会用各种方式表达同一概念）：
   ```yaml
   alias: [渠道, 获客渠道, 放款渠道, 来源渠道, channel]
   ```

3. **examples 要包含真实枚举值**（影响value召回准确率）：
   ```yaml
   examples: [自营APP, 京东金融, 抖音, 线下门店-北京, 线下门店-上海]
   ```

4. **layer 字段要正确标注**（影响分层路由）：
   ```yaml
   layer: ADS  # 或 DWD
   ```

5. **歧义指标要加标记**：
   ```yaml
   ambiguous: true  # 只给逾期率这类有多种口径的指标加
   ```
