# 信贷业务指标定义

> 供 B3（meta_config.yaml 的 metrics 部分）参考。
> description 字段直接影响 LLM 能否生成正确 SQL，要写清楚计算公式。

---

## 指标清单

### 放款类指标

| 指标名 | alias | 计算公式 | relevant_columns | ambiguous |
|--------|-------|---------|-----------------|-----------|
| 放款金额 | 放款额/放款总额/发放金额 | SUM(loan_amount) | dwd_loan.loan_amount | false |
| 放款笔数 | 放款量/放款单量/新增贷款笔数 | COUNT(loan_id) | dwd_loan.loan_id | false |
| 平均放款金额 | 件均放款/平均件均/平均贷款金额 | AVG(loan_amount) | dwd_loan.loan_amount | false |

**description 示例（放款金额）**：
```
当期所有放款记录的loan_amount总和，反映业务规模。
按月统计时用ads_loan_monthly.loan_amount，按渠道/产品细分时加GROUP BY。
```

### 逾期类指标

| 指标名 | alias | 计算公式 | relevant_columns | ambiguous |
|--------|-------|---------|-----------------|-----------|
| 逾期率 | 违约率 | 逾期金额/应还金额（需澄清口径） | dwd_overdue.overdue_amount, dwd_repayment.due_amount | **true** |
| 首逾率(M1) | M1逾期率/首次逾期率 | M1逾期笔数/当期到期笔数×100% | dwd_overdue.bucket, dwd_repayment.due_count | false |
| 在贷逾期率 | 存量逾期率 | 当前逾期余额/当前在贷余额×100% | dwd_overdue.overdue_amount, dwd_loan.loan_amount | false |
| 平均逾期天数 | 平均账龄/平均拖欠天数 | AVG(overdue_days)，仅统计逾期记录 | dwd_overdue.overdue_days | false |

**description 示例（逾期率）**：
```
逾期率有多种口径，需要用户澄清：
- 首逾率(M1)：当月新增M1逾期笔数/当月到期应还笔数，通常约2-4%
- 在贷逾期率：当前时点逾期余额/在贷余额，通常约3-6%
- 整体逾期率：历史累计逾期金额/历史累计放款金额
使用ads_overdue_monthly表可直接获取各口径已计算值。
```

**description 示例（首逾率M1）**：
```
首逾率(M1)：当月到期还款中，首次逾期（逾期桶=M1）的笔数占当月到期总笔数的比例。
计算：ads_overdue_monthly.m1_overdue_rate（已算好），或从dwd_overdue过滤bucket='M1'计算。
正常区间约2-4%，超过6%需关注。
```

|--------|-------|---------|-----------------|-----------|

## 写 description 的模板

每个指标的 description 应包含：

```
1. 一句话定义（是什么）
2. 计算公式（怎么算）
3. 推荐使用的表（从哪里查）
4. 正常范围或业务含义（可选）
```

例：
```yaml
- name: 放款笔数
  description: >
    当期新增放款的总笔数，反映业务量规模。
    计算：COUNT(dwd_loan.loan_id)，或直接用ads_loan_monthly.loan_count（ADS层已汇总）。
    按渠道分析时GROUP BY channel_name，按产品分析时GROUP BY product_name。
  relevant_columns:
    - dwd_loan.loan_id
    - ads_loan_monthly.loan_count
  alias: [放款量, 放款单量, 新增贷款笔数, 新增放款量, 放款数量]
  ambiguous: false
```
