## 详细笔记

### Q5：retrieved_values 的取值如何和字段关联

**核心机制**：`ValueInfo.column_id` 字段

```
ValueInfo(value="自营APP", column_id="fact_loan.channel")
                    ↓
        通过 column_id 找到对应字段
                    ↓
        追加到 ColumnInfo.examples 列表
```

### Q12：TypedDict vs dataclass vs dict

| 方案 | 运行时类型 | 访问方式 | LangGraph 兼容 |
|------|-----------|---------|---------------|
| TypedDict | dict | state["key"] | ✅ 完全兼容 |
| dataclass | 对象 | obj.key | ❌ 不兼容 |
| 普通 dict | dict | state["key"] | ✅ 兼容但无类型提示 |

---

### Q16：单例模式 + 连接池

- **初始化时机**：服务启动时（lifespan），不是用户第一次请求
- **不是单个连接**：是连接池（pool_size=10）
- **共享**：多个请求复用连接池

---

### Q22：SSE 流式响应

```
用户请求 → FastAPI → StreamingResponse
                        ↓
              graph.astream() 异步生成器
                        ↓
              节点通过 stream_writer 推送
                        ↓
              包装成 SSE 格式：data: {json}\n\n
                        ↓
              浏览器逐帧接收
```

---

### Q23：stream_writer 传入机制

- **不是在节点里初始化**：在 QueryService 启动图时创建
- **通过 runtime 传入**：`writer = runtime.stream_writer`
- **不用 State 的原因**：State 是业务数据，writer 是基础设施对象

---

### Q24：依赖注入 + Mock

- **依赖注入**：通过 `Depends(工厂函数)` 自动组装依赖
- **Mock**：测试时用假对象替换真实组件，方便测试

---

### Q26：build_meta_knowledge.py

```
meta_config.yaml → Meta MySQL（结构化）
                 → Qdrant（向量索引）
                 → ES（枚举值索引）
```

---

### Q36：共享 vs 隔离

| 组件 | 共享/隔离 | 原因 |
|------|----------|------|
| graph | 共享（只读） | 流程定义，不复制 |
| state | 隔离 | 每个请求数据不同 |
| 连接池 | 共享 | 复用有限资源 |
