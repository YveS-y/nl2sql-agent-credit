# 项目启动流程

## 停止服务

### 正常停止（数据保留，下次可直接启动）

```bash
docker-compose stop
```

Embedding / 后端 / 前端：在各自终端按 `Ctrl+C` 即可。

### 区别说明

| 命令 | 容器 | 数据卷 | 适合场景 |
|------|------|--------|---------|
| `docker-compose stop` | 停止 | ✅ 保留 | 日常暂停 |
| `docker-compose down` | 删除 | ✅ 保留 | 彻底清理容器但保留数据 |
| `docker-compose down -v` | 删除 | ❌ 删除 | 重置所有数据（慎用） |

---

## 每次开发前按顺序执行

### 第一步：启动基础设施（Docker）

```bash
docker-compose up -d mysql qdrant elasticsearch
```

等 MySQL healthy（约15秒）后再继续。

> 注意：不要用 `docker-compose up -d`（不带参数），那会构建 data-agent 镜像，非常慢且没必要——后端直接在宿主机跑。

### 第二步：启动 Embedding 服务（新开终端）

```bash
uv run uvicorn docker.embedding.server:app --port 8081
```

第一次启动会下载模型（BAAI/bge-large-zh-v1.5），需要几分钟。后续启动很快。

> 不要用 Docker 镜像 `ghcr.io/huggingface/text-embeddings-inference`，它是 AMD64，M4 跑不了。

### 第三步：启动后端（新开终端）

```bash
uv run uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 第四步：启动前端（新开终端）

```bash
cd data-agent-fronted/data-agent-fronted && npm run dev
```

访问 http://localhost:5173

## 重建数据（数据丢失时）

```bash
# 1. 清空并重建数据卷
docker-compose down -v && docker-compose up -d mysql qdrant elasticsearch

# 2. 生成信贷模拟数据（30000条）
uv run --with pymysql python scripts/generate_credit_data.py

# 3. 重建元数据索引（需要 embedding 服务已启动）
PYTHONPATH=. uv run python app/scripts/build_meta_knowledge.py -c conf/meta_config.yaml
```
