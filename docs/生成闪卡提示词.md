# 生成项目闪卡的提示词模板

> 复制下面的提示词，粘贴到新项目的对话里，让 Claude 生成该项目的闪卡。

---

## 提示词

```
我正在学习一个新项目，请帮我生成一份适合手机端碎片化复习的闪卡文档。

要求：
1. 格式为 Q&A（一问一答），每个知识点独立成一条
2. 答案控制在5行以内，手机屏幕一屏能看完
3. 覆盖以下模块（根据项目实际情况调整）：
   - 整体架构：项目是干什么的、核心组件有哪些、数据怎么流动
   - 核心流程：主要处理步骤/节点，每步输入输出是什么
   - 数据模型：有哪些表/实体，关键字段，表之间的关系
   - 核心概念：项目特有的业务术语、设计决策、规则
   - 易混淆点：容易搞混的概念，用对比方式呈现
   - 关键代码位置：重要功能在哪个文件/类/函数里

4. 每条 Q&A 之间用 --- 分隔
5. 按模块分组，每个模块有二级标题
6. 问题要像自测题，答案要像标准答案，不废话

请先阅读项目的关键文件（README、核心代码、配置文件），然后生成闪卡。
生成完后，把文件保存到 docs/项目闪卡.md。
```

---

## 使用说明

1. 打开新项目，开始一个新的对话
2. 先让 Claude 读项目文件（`@README` 或指定目录）
3. 再把上面的提示词发给 Claude
4. 生成后保存到 `docs/项目闪卡.md`

## 手机端查看方式

- **GitHub**：直接在手机浏览器打开仓库，点 `docs/项目闪卡.md`，GitHub 会渲染 Markdown
- **备忘录**：把内容复制到手机备忘录，格式稍乱但能看
- **Obsidian**（推荐）：把项目文件夹同步到 iCloud，用 Obsidian 手机版打开，支持 Markdown 渲染和闪卡插件

## 提升效果的技巧

- 遮住答案，只看问题，默想答案后再翻开对比
- 每天只背 5-10 条，反复循环比一次背完效果好
- 答错的条目用 ⭐ 标记，下次优先复习
